# Идеи по доработке проекта (Binary Options)

Анализ проекта **TradingNew** — платформа бинарных опционов с Next.js, Express, Prisma, PostgreSQL, HighHelp, реферальной программой и админкой. Ниже перечислено, **чего ещё можно добавить** с разбивкой по приоритету и сложности.

---

## Уже реализовано (кратко)

- Регистрация/логин, JWT, cookie, сессии в БД, 2FA (TOTP)
- Торговля: пары, LONG/SHORT, экспирация, расчёт WIN/LOSS на сервере
- Реальные цены: Binance REST + WebSocket (не mock)
- OHLC-свечи, таймфреймы
- Пополнение/вывод через HighHelp (P2P, RUB), webhook, идемпотентность
- Реферальная программа (внутри ЛК + отдельный домен referral-site)
- Партнёрская программа (ReferralPartner): CPA, вывод, аналитика
- Админка: пользователи, блокировки, баланс, пары, настройки, поддержка
- Поддержка: треды, сообщения user/admin
- Аудит баланса (BalanceAuditLog), Telegram-уведомления и алерты по аномалиям
- Rate limit на trade/open, deposit/withdraw, реферальный вывод
- Блокировки: полная (торги+вывод) и только вывод
- Мультиязычность (i18n), мобильная навигация, лендинг

## Функции именно по бинарным опционам

Идеи, которые касаются только механики торговли опционами: открытие/закрытие, график, экспирация, отображение выигрыша, риск, история сделок.

### Открытие сделки и панель ордера

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Потенциальный выигрыш до клика** | Показывать под кнопками LONG/SHORT: «При выигрыше: +$X» (ставка × (1 + winPayoutPercent/100)). Нужен эндпоинт `GET /me` или отдельный `GET /trading/config` с полем `winPayoutPercent` для фронта (сейчас процент только в админке). | Низкая |
| **Быстрые пресеты суммы** | Кнопки/чипы: $10, $25, $50, $100, «50% баланса», «Макс» — подстановка в поле суммы без ввода вручную. | Низкая |
| **Быстрые пресеты экспирации** | Кнопки: 1 мин, 2 мин, 5 мин, 15 мин (сейчас только число в секундах; бэкенд уже принимает durationSeconds, минимум 60). | Низкая |
| **Турбо-опционы (30 сек)** | Разрешить экспирацию 30 сек (или 15/45) в дополнение к текущему минимуму 60 сек. На бэкенде — уменьшить проверку `durationSeconds >= 60` до `>= 30` (или вынести минимум в AppSetting). На фронте — пресет «30 с» и валидация. | Низкая |
| **Повторить последнюю сделку** | Кнопка «Повторить»: те же пара, сумма, экспирация, направление (или только сумма/время, направление выбирает пользователь). Сохранять в store последние параметры ордера. | Низкая |
| **Подтверждение при большой ставке** | Модалка «Вы уверены? Ставка $X (Y% баланса)» при сумме выше порога (например >20% баланса или >$100). Порог — в настройках или константа. | Низкая |
| **Лимит ставки от баланса** | Ограничение: не более N% баланса на одну сделку (настройка в админке или константа). Проверка на бэкенде при `POST /trade/open` + подсказка на фронте. | Низкая |

### График и анализ

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Маркер выхода на графике** | Показывать на графике не только точку входа (entry), но и цену на момент экспирации (closePrice) для завершённых сделок — второй маркер «exit» с цветом WIN/LOSS. Данные уже есть в `Trade.closePrice`. | Низкая |
| **Вертикальная линия экспирации** | Для активной сделки — вертикальная линия на графике в момент `expiresAt`, чтобы визуально видеть «до когда» идёт опцион. | Низкая |
| **Разный payout по экспирации** | Разный процент выигрыша в зависимости от срока: например 85% для 30 сек, 90% для 1 мин, 95% для 5 мин. На бэкенде — при settlement брать процент из конфига по `durationSeconds`; в админке — настройка «сетка» (секунды → процент). | Средняя |
| **Область «в плюсе/в минусе»** | Для активной сделки подсвечивать на графике зону выше/ниже entry price (зелёная — выигрыш для LONG, красная — проигрыш и наоборот для SHORT). | Средняя |
| **Таймфрейм «под экспирацию»** | Если выбран срок 1 мин — предлагать или автоматически подставлять таймфрейм графика 1m (или 30s для турбо), чтобы свечи совпадали с длительностью опциона. | Низкая |

### Типы графиков (режимы отображения цены)

Сейчас в проекте два режима: **Линия** (реализована через `AreaSeries` — линия + заливка под кривой) и **Свечи** (`CandlestickSeries`). Библиотека **lightweight-charts (v5)** поддерживает ещё несколько типов серий, которые можно добавить как отдельные режимы.

| Режим | Описание | Библиотека | Сложность |
|-------|----------|------------|-----------|
| **Линия (тонкая)** | Классическая линия по close без заливки — минималистичный вид. | `LineSeries` | Низкая |
| **Area (уже есть как «Линия»)** | Линия + градиентная заливка под кривой. Сейчас это текущий режим «Линия». | `AreaSeries` | — |
| **Baseline** | Линия относительно базового уровня (например первая точка или скользящая средняя): зона выше базовой — один цвет (например зелёный), ниже — другой (красный). Удобно для бинарных опционов: наглядно «в плюсе / в минусе» относительно точки входа. | `BaselineSeries` | Низкая |
| **Свечи (уже есть)** | Японские свечи OHLC. | `CandlestickSeries` | — |
| **Бары (OHLC)** | Столбики с Open/High/Low/Close без «тела» свечи — только вертикальная линия и горизонтальные черточки. Более компактный вариант свечей. | Можно эмулировать через `CandlestickSeries` с тонким телом или отдельной отрисовкой; в v5 нет отдельного BarSeries — обычно используют свечи. | Средняя |
| **Heikin Ashi** | Свечи с пересчитанными OHLC (сглаженный вид, меньше шума). Данные считаются из обычных OHLC на фронте или бэкенде. | `CandlestickSeries` с преобразованными данными | Средняя |
| **Объём под ценой** | Гистограмма объёма под основным графиком (если будет источник объёма по таймфреймам). | `HistogramSeries` на отдельной панели | Средняя |

**Рекомендация:** быстрее всего добавить **Baseline** (базовая линия = первая точка или последняя цена входа по активной сделке) и **Line** в смысле чистой `LineSeries` без заливки — оба есть в lightweight-charts, данные те же (close или OHLC).

### Закрытие и результат

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Раннее закрытие (sell option)** | Досрочно закрыть опцион до экспирации с частичным возвратом (например 30–50% ставки). Нужна логика на бэкенде: новая операция «early close», пересчёт и обновление баланса, статус сделки. Не все платформы это делают — по желанию продукта. | Высокая |
| **Итог по сделке в списке** | В блоке активных/завершённых сделок показывать не только WIN/LOSS, но и «+$X» / «−$X» (сумма выигрыша или потери). Данные считаются: при WIN — payout − stake, при LOSS — −stake. | Низкая |
| **Звук при закрытии** | Включённый в настройках звук при смене статуса сделки на WIN или LOSS (уже в общем списке идей; для опционов особенно важно — пользователь ждёт экспирации). | Низкая |
| **Вибрация на мобильном** | При закрытии сделки — короткая вибрация (navigator.vibrate), разная для WIN/LOSS. | Низкая |

### История и статистика по сделкам

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Фильтры истории** | По паре, по направлению (LONG/SHORT), по результату (WIN/LOSS), по дате. Бэкенд: query-параметры в `GET /trades/completed`; фронт — выпадающие фильтры на странице истории. | Низкая |
| **Статистика по парам** | В профиле или на странице торговли: «По BTCUSDT: 12 WIN, 8 LOSS, +$45» (сумма выигрышей минус проигрышей по этой паре). Эндпоинт `GET /trades/stats-by-pair` или агрегация на фронте по completed trades. | Низкая |
| **Винрейт и P&amp;L** | Общий винрейт (WIN / (WIN+LOSS)) и общий P&amp;L по завершённым сделкам — блок в профиле или над историей. | Низкая |
| **График P&amp;L по дням** | Кривая «баланс по дням» или «прибыль/убыток по дням» на основе сделок (или BalanceAuditLog). Отдельная страница или виджет в профиле. | Средняя |

### Риск и лимиты (только торги)

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Макс. ставка от баланса** | Не давать открыть сделку, если ставка > N% текущего баланса (настраиваемый лимит в админке). | Низкая |
| **Дневной лимит убытков (опционально)** | Ограничение: если убыток за день превысил X — блокировать новые сделки до следующего дня. Нужно считать дневной P&amp;L по сделкам и проверять при открытии. | Средняя |
| **Макс. число активных сделок** | Лимит одновременно открытых опционов (например 5 или 10). Проверка при `POST /trade/open`. | Низкая |

### Типы опционов (расширение продукта)

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Touch / No Touch** | Опцион «цена коснётся уровня X до экспирации» (Touch) или «не коснётся» (No Touch). Нужна новая модель/поля (тип опциона, целевая цена), логика settlement по тикам/свечам. | Высокая |
| **Boundary (In/Out)** | Опцион «цена останется внутри диапазона [A, B]» (In) или «выйдет за диапазон» (Out). Аналогично — новый тип, расчёт по OHLC за период. | Высокая |
| **Ладдер (несколько экспираций)** | Одна «ставка» с несколькими экспирациями подряд (например 1м, 2м, 5м): выигрыш по каждому сроку отдельно. Усложняет модель Trade и settlement. | Высокая |

Текущая реализация — классические **High/Low (LONG/SHORT)** с одной экспирацией и одной ценой закрытия; всё выше — дополнения к этой модели.

## 1. Безопасность и устойчивость

| Идея | Описание | Сложность |
|------|----------|-----------|
| **Rate limit на логин/регистрацию** | Ограничение попыток входа и регистрации по IP (например, 5–10 в минуту), как указано в SECURITY_ANALYSIS.md. Снижает брутфорс и спам-регистрации. | Низкая |

| **Ограничение доступа к админке** | Доп. защита: whitelist IP, отдельный поддомен или VPN. | Низкая |

## Приоритизация (рекомендации)

**Быстрые победы (низкая сложность, высокая польза):**

1. Rate limit на `/auth/login` и `/auth/register`.
2. Фильтры и сортировка в истории сделок.
3. Минимальная/максимальная сумма вывода (настройка + проверка).
4. Поиск пользователей в админке.
5. FAQ-страница и чекбокс оферты при регистрации.

**Средний приоритет:**
