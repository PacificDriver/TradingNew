# Анализ безопасности и запуск в production

Сервис работает с реальными деньгами (пополнение/вывод через HighHelp). Ниже — сводка по безопасности и обязательные шаги перед production.

---

## 1. Аутентификация и сессии

| Аспект | Статус | Комментарий |
|--------|--------|-------------|
| Пароли | ✅ | Хеширование bcrypt (10 раундов при регистрации/смене пароля). |
| JWT | ⚠️ | В проде **обязательно** задать свой `JWT_SECRET` (длинный случайный). По умолчанию `supersecretjwtkey` — недопустимо в production. |
| Срок жизни токена | ✅ | 7 дней. Сессии хранятся в БД, logout инвалидирует сессию. |
| Cookie | ✅ | `httpOnly`, `sameSite: lax`, `secure` в production (`IS_PROD`). |
| 2FA (TOTP) | ✅ | Опционально для пользователей. |
| Админ | ✅ | Отдельный флаг `isAdmin`, админ-маршруты защищены `adminMiddleware`. |

**Риски:** Нет rate limiting на `/auth/login` и `/auth/register` — возможен перебор паролей и массовая регистрация. Рекомендуется добавить лимиты по IP (например, 10 попыток входа в минуту).

---

## 2. Платежи и баланс

| Аспект | Статус | Комментарий |
|--------|--------|-------------|
| Источники пополнения | ✅ | Только колбек HighHelp (payin success), возврат при отказе вывода, выигрыш по сделке, перевод реферального баланса. |
| Проверка колбека | ✅ | `project_id` + опционально RSA (`HIGHHELP_CALLBACK_PUBLIC_KEY_PEM`) или секрет/HMAC (`HIGHHELP_WEBHOOK_SECRET`). В проде **обязательно** задать публичный ключ колбеков. |
| Идемпотентность колбеков | ✅ | `ProcessedCallback` по ключу `project_id:payment_id:status:sub_status`. |
| Гонки при списании/начислении | ✅ | Атомарные UPDATE (сделка, вывод, реферальный перевод), расчёт экспирации только по сделкам в статусе ACTIVE. |
| Rate limiting | ✅ | 30 запросов/мин на открытие сделки, 10/мин на пополнение/вывод/реферальный вывод. |
| Аудит баланса | ✅ | Таблица `BalanceAuditLog` на каждое изменение. |
| Уведомления и аномалии | ✅ | Опционально Telegram (каждое изменение + алерты по аномалиям). |

---

## 3. API и данные

| Аспект | Статус | Комментарий |
|--------|--------|-------------|
| Авторизация | ✅ | Все операции с балансом/торгами/выводом требуют JWT. `userId` из токена, не из тела. |
| Блокировки | ✅ | `requireNotBlockedMiddleware`, `withdrawBlockedAt` проверяются где нужно. |
| SQL-инъекции | ✅ | Prisma + параметризованные запросы; raw-запросы через шаблонные литералы с подстановкой. |
| CORS | ⚠️ | В проде **не** использовать `CORS_ALLOW_ANY=1`. Задать явный `FRONTEND_ORIGIN` (через запятую при нескольких доменах). |
| Утечка данных | ✅ | В ответах не отдаются пароли, TOTP-секреты; ошибки для пользователя — общие (коды E001/E002/E003 в консоли для админа). |

---

## 4. Инфраструктура и конфигурация

| Аспект | Статус | Комментарий |
|--------|--------|-------------|
| HTTPS | ⚠️ | В production бэкенд и фронт должны работать только по HTTPS. Прокси (nginx/cloud) должен терминировать TLS. |
| Секреты | ⚠️ | Все секреты только в `.env`, не в репозитории. `.env` в `.gitignore`. |
| База данных | ⚠️ | Надёжный `DATABASE_URL`, ограничение доступа по сети, регулярные бэкапы. |
| Логи | ✅ | В production `removeConsole` оставляет только `error`/`warn`. Не логировать пароли, токены, полные тела запросов с картами. |

---

## 5. Что остаётся на усмотрение

- **Rate limiting на логин/регистрацию** — снизить риск брутфорса и спам-регистраций.
- **Мультиинстанс:** текущий rate limit — in-memory; при нескольких инстансах бэкенда нужен общий store (например Redis).
- **Доп. защита админки:** ограничение по IP, VPN или отдельный домен.
- **Срок жизни сессии:** при необходимости сократить с 7 дней (например до 1 дня в проде).

## 6. Чек-лист перед запуском в production

Выполните по порядку перед тем, как запускать сервис с реальными деньгами.

### Секреты и переменные (backend `.env`)

- [ ] **JWT_SECRET** — задан свой длинный случайный секрет (минимум 32 символа), не дефолтный.
- [ ] **DATABASE_URL** — указана production БД, доступ ограничен по сети.
- [ ] **HIGHHELP_PROJECT_ID**, **HIGHHELP_PRIVATE_KEY_PEM**, **BACKEND_PUBLIC_URL** — заданы и соответствуют ЛК HighHelp.
- [ ] **HIGHHELP_CALLBACK_PUBLIC_KEY_PEM** — публичный ключ из ЛК HighHelp (API → Настройки Callback → Public Key). Без него колбеки проверяются только по `project_id`.
- [ ] **BACKEND_PUBLIC_URL** — только HTTPS, например `https://api.yourdomain.com`.
- [ ] **FRONTEND_ORIGIN** — явный список разрешённых Origin (без `CORS_ALLOW_ANY=1` в проде).
- [ ] Файл `.env` не попадает в git (проверить `.gitignore`).

### HighHelp и платежи

- [ ] В ЛК HighHelp в настройках кассы указан URL колбека: `{BACKEND_PUBLIC_URL}/payments/webhook`.
- [ ] Проведена тестовая оплата и проверка прихода колбека и начисления баланса (и при необходимости — возврат при отказе вывода).

### Фронтенд

- [ ] Сборка для production: `npm run build` в `frontend/`.
- [ ] В production заданы **NEXT_PUBLIC_API_BASE_URL** и **NEXT_PUBLIC_WS_URL** на ваш бэкенд (HTTPS/WSS), если фронт и бэкенд на разных доменах.

### Инфраструктура

- [ ] Бэкенд и фронт доступны только по **HTTPS** (TLS на прокси/балансировщике).
- [ ] Настроены бэкапы БД и их проверка восстановления.
- [ ] Логи приложения не пишут пароли, токены, полные номера карт и т.п.

### Админ и первый запуск

- [ ] Админ создан через `npx ts-node scripts/create-admin.ts` (пароль из `ADMIN_PASSWORD` в `.env` или сгенерирован и сохранён).
- [ ] После первого входа админа пароль от скрипта сменён в интерфейсе при необходимости.

### Мониторинг (рекомендуется)

- [ ] Настроены уведомления в Telegram (TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID) для изменений баланса и аномалий.
- [ ] Мониторинг доступности бэкенда (health check: `GET /health`) и БД.

## 7. Быстрая сводка

| Категория | Критично в проде |
|-----------|-------------------|
| Секреты | JWT_SECRET свой; HIGHHELP_* и BACKEND_PUBLIC_URL заданы; проверка колбеков (публичный ключ). |
| Сеть | HTTPS; CORS только на свои Origin. |
| Данные | Бэкапы БД; не логировать чувствительные данные. |
| Платежи | Успешный тест пополнения/вывода и колбека. |

После выполнения чек-листа сервис можно запускать в production с учётом рисков, описанных в разделах 1–5.
